# 2.1 Data ingestion and acquisition

## Exam objective

> Design and implement data ingestion pipelines to efficiently ingest a variety of data formats including Delta Lake, Parquet, ORC, AVRO, JSON, CSV, XML, Text and Binary from diverse sources such as message buses and cloud storage.
>
> Create an append-only data pipeline capable of handling both batch and streaming data using Delta.

## Must know

- Ingestion must land in **Unity Catalog-governed** Delta tables for production (`catalog.schema.table`).
- **Append-only** pipelines add new records without updating history in bronze; merges/CDC happen in silver+.
- **Auto Loader** (`cloudFiles`) is the default for scalable file ingestion from cloud storage or volumes.
- **Structured Streaming** and **SDP streaming tables** read message buses (Kafka, Kinesis, Pub/Sub, etc.).
- **Lakeflow Connect** covers managed SaaS/database connectors when the source is not raw files.
- Format choice affects schema handling: Parquet/ORC/Avro/Delta are schema-aware; JSON/CSV/XML often need inference or explicit schema.
- **Binary** files usually need custom parsing or metadata extraction before tabular modeling.
- Same Delta table can serve **batch** and **streaming** writers when operations are compatible (append vs conflicting overwrites).

## Format ingestion matrix

| Format | Typical read API | Schema | Professional note |
|--------|------------------|--------|-------------------|
| Delta | `spark.table()` / `read.format("delta")` | Enforced by table | Preferred curated storage |
| Parquet | `read.parquet()` | Embedded | Good bronze landing; convert to Delta for ACID |
| ORC | `read.format("orc")` | Embedded | Similar to Parquet |
| Avro | `read.format("avro")` | Avro schema | Common in Kafka pipelines |
| JSON | `read.json()` / Auto Loader | Infer or explicit | Use `schemaLocation`, `rescuedDataColumn` |
| CSV | `read.csv()` / Auto Loader | Infer or explicit | Header, delimiter, multiline options |
| XML | `read.format("xml")` | Row tag / XSD | Spark XML datasource; define row tag |
| Text | `read.text()` | Single `value` column | Logs, lines; parse downstream |
| Binary | `read.format("binaryFile")` | path, length, content | Images/PDFs; often to volumes + metadata table |

## Source type patterns

```
Message bus (Kafka, etc.)
  -> Structured Streaming / SDP streaming table
  -> bronze append Delta table

Cloud storage / UC volume (files)
  -> Auto Loader (cloudFiles)
  -> bronze append Delta table

SaaS / RDBMS
  -> Lakeflow Connect or JDBC batch
  -> bronze Delta table
```

## Append-only batch + streaming on Delta

| Mode | Write pattern | Checkpoint |
|------|---------------|------------|
| Batch append | `mode("append")` or `INSERT` | Not required for one-shot batch |
| Streaming append | `writeStream.format("delta").outputMode("append")` | Required |
| Auto Loader micro-batch | `Trigger.AvailableNow` or processing time | Required |

```python
# Streaming append to UC Delta table
(
    spark.readStream.format("cloudFiles")
    .option("cloudFiles.format", "json")
    .load("/Volumes/main/landing/orders/")
    .writeStream
    .format("delta")
    .outputMode("append")
    .option("checkpointLocation", "/Volumes/main/checkpoints/orders_bronze")
    .table("main.bronze.orders_raw")
)
```

```python
# Batch append (same target table — coordinate scheduling)
(
    spark.read.format("json")
    .load("/Volumes/main/landing/orders/batch_2026_05_23/")
    .write.format("delta")
    .mode("append")
    .saveAsTable("main.bronze.orders_raw")
)
```

Exam rule: bronze is **append-only** unless the question explicitly asks for overwrite/CDC at bronze.

## When to use what

| Requirement clue | Best choice |
|------------------|-------------|
| Millions of small files arriving daily | Auto Loader |
| Kafka topic with Avro schema | Structured Streaming or SDP + schema registry |
| One-time bulk Parquet migration | Batch read + `saveAsTable` or `COPY INTO` |
| XML enterprise feed | `read.format("xml")` with rowTag; land bronze then parse |
| PDF/images with governance | UC volume + binaryFile + metadata Delta table |
| SQL-only scheduled file load, low volume | `COPY INTO` |
| Salesforce/Workday managed sync | Lakeflow Connect |

## PySpark / API essentials

```python
# Parquet batch to Delta bronze
(
    spark.read.parquet("/Volumes/main/landing/sales/")
    .write.format("delta")
    .mode("append")
    .saveAsTable("main.bronze.sales_raw")
)

# Kafka streaming (conceptual)
(
    spark.readStream.format("kafka")
    .option("kafka.bootstrap.servers", "broker:9092")
    .option("subscribe", "orders")
    .load()
    .selectExpr("CAST(value AS STRING) AS body")
    .writeStream
    .format("delta")
    .outputMode("append")
    .option("checkpointLocation", "/Volumes/main/checkpoints/kafka_orders")
    .table("main.bronze.orders_kafka")
)

# Binary files
binary_df = spark.read.format("binaryFile").load("/Volumes/main/docs/")
```

## SQL (exam)

```sql
-- COPY INTO for controlled batch file loads
COPY INTO main.bronze.events_raw
FROM '/Volumes/main/landing/events/'
FILEFORMAT = JSON
FORMAT_OPTIONS ('inferSchema' = 'true')
COPY_OPTIONS ('mergeSchema' = 'true');

-- SQL streaming table with Auto Loader (SDP)
CREATE OR REFRESH STREAMING TABLE main.bronze.events_raw
AS SELECT * FROM STREAM read_files(
  '/Volumes/main/landing/events/',
  format => 'json'
);
```

## UI / product surfaces

- **Catalog Explorer**: verify bronze table growth, schema, properties.
- **Lakeflow Pipelines / Jobs**: schedule ingestion, monitor failures.
- **Auto Loader metrics**: files discovered vs processed in pipeline event log.

## Exam traps

- Overwriting bronze on every run when question says append-only audit trail.
- Using `COPY INTO` for continuous high-volume file arrival — Auto Loader scales better.
- Forgetting checkpoint location uniqueness per stream.
- Reading binary files as CSV/JSON without `binaryFile` format.
- Mixing incompatible write modes (complete output mode on keyed tables without clear requirement).
- Ingesting to DBFS root instead of UC volumes/tables for new governed workloads.
