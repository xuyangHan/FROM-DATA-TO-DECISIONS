# 7.1 Applying data security mechanisms

## Exam objective

> Use ACLs to secure Workspace Objects, enforcing the principle of least privilege, including enforcing principles like least privilege, policy enforcement.
>
> Use row filters and column masks to filter and mask sensitive table data.
>
> Apply anonymization and pseudonymization methods, such as Hashing, Tokenization, Suppression, and generalisation, to confidential data.

## Must know

- **Least privilege**: grant minimum permissions needed; prefer groups over individual users.
- **Workspace ACLs** control notebooks, folders, clusters, jobs, SQL objects outside UC scope.
- **Unity Catalog** controls data securables (`SELECT`, `MODIFY`, `CREATE`, `MANAGE`, etc.).
- **Row filters** restrict which rows a principal sees via boolean UDF at fetch time.
- **Column masks** transform column values via UDF at fetch time.
- **Anonymization** removes/replaces identifying info; **pseudonymization** replaces with reversible token (separate vault).
- Techniques: **hashing**, **tokenization**, **suppression** (redact), **generalization** (bucket ages, truncate ZIP).

## Workspace ACLs vs Unity Catalog

| Layer | Secures | Example |
|-------|---------|---------|
| Workspace ACL | Non-UC or workspace-local objects | Notebook, folder, legacy job |
| Unity Catalog | Tables, views, volumes, functions | `GRANT SELECT ON TABLE main.silver.orders` |
| Cluster policies | Compute guardrails | Instance types, autoscale limits |
| Account groups | Identity | Map SSO groups to UC groups |

Least privilege pattern:

```sql
GRANT USE CATALOG ON CATALOG main TO `analysts`;
GRANT USE SCHEMA ON SCHEMA main.gold TO `analysts`;
GRANT SELECT ON TABLE main.gold.daily_revenue TO `analysts`;
-- No MODIFY, no CREATE, no ALL PRIVILEGES unless required
```

## Row filters and column masks

```sql
CREATE FUNCTION main.security.mask_email(email STRING)
RETURN CASE
  WHEN is_account_group_member('hr_team') THEN email
  ELSE regexp_replace(email, '(?<=.).(?=.*@)', '*')
END;

ALTER TABLE main.silver.customers
ALTER COLUMN email SET MASK main.security.mask_email;

CREATE FUNCTION main.security.region_filter(region STRING)
RETURN is_account_group_member('global_analysts') OR region = 'EU';

ALTER TABLE main.silver.customers
SET ROW FILTER main.security.region_filter ON (region);
```

| Mechanism | UDF return type | Effect |
|-----------|-----------------|--------|
| Column mask | Same as column (or castable) | Masks values in `SELECT` |
| Row filter | `BOOLEAN` | Hides rows where `FALSE`/`NULL` |

Masks/filters do **not** grant access â€” user still needs `SELECT`.

## Anonymization toolkit

| Method | Description | Reversible? | Exam example |
|--------|-------------|-------------|--------------|
| Hashing | One-way fingerprint (`sha2`, `md5`) | No | Share join keys without raw email |
| Tokenization | Replace with token; map in secure vault | Yes (with vault) | Payment card token |
| Suppression | Remove/redact field (`NULL`, `***`) | No | Drop SSN column for analysts |
| Generalization | Reduce precision | No | Age â†’ age band; ZIP â†’ first 3 digits |

```python
from pyspark.sql.functions import sha2, col, lit, floor, concat

masked = (
    df.withColumn("email_hash", sha2(col("email"), 256))
    .withColumn("age_band", floor(col("age") / 10) * 10)
    .drop("email", "ssn")  # suppression
)
```

| Requirement | Prefer |
|-------------|--------|
| Need stable join key without revealing PII | Hashing with salt (managed) |
| Partner must never see raw PAN | Tokenization + vault |
| Public aggregate release | Generalization + suppression |
| Dynamic per-role visibility in SQL | UC column mask / row filter |
| Many tables with `pii` tag | ABAC policies (see Associate 7.4) |

## When to use what

| Requirement clue | Best choice |
|------------------|-------------|
| Contractors see customers but not salary | Column mask on `salary` |
| EU team sees only EU rows | Row filter on `region` |
| Remove direct identifiers before gold | Suppression + hashing in silver pipeline |
| Publish coarse demographics only | Generalization |
| Lock down notebook access | Workspace ACL on folder |

## PySpark / API essentials

Prefer UC masks for query-time enforcement; pipeline transforms for irreversible anonymization before broader sharing.

## SQL (exam)

Row filter / column mask DDL above; `DENY` overrides grants when exceptions required.

## UI / product surfaces

- **Catalog Explorer â†’ Permissions**: grants, effective permissions.
- **Workspace â†’ Share / Permissions**: notebook/folder ACLs.
- **Admin â†’ Groups**: group membership for `is_account_group_member()`.

## Exam traps

- Granting `ALL PRIVILEGES` when `SELECT` suffices.
- Column mask without base `SELECT` â€” users still need table access.
- Reversible token stored in same table as tokenized column.
- Hash without salt for passwords/PII (exam expects awareness of weakness).
- Row filter UDF returning string instead of boolean.
- Using dynamic views when centralized tag-based policy (ABAC) scales better.
