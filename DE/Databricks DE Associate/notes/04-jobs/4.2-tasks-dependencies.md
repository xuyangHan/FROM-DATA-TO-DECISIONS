# 4.2 Job tasks & DAG dependencies

## Exam objective

> Configure common tasks (notebook, SQL query, dashboard, and pipeline tasks) and their dependencies using Lakeflow Jobs and its DAG-based task graph.

## Must know

- **Notebook task**: runs `.ipynb` or workspace path; parameters via widgets.
- **SQL task**: runs SQL file or warehouse query; needs SQL warehouse or SQL-capable endpoint.
- **Dashboard task**: refreshes Lakeview/AI/BI dashboard.
- **Pipeline task**: runs Lakeflow Spark Declarative Pipeline (LDP).
- **DAG**: directed acyclic graph — downstream starts after upstream **success** (default).
- **Task values**: pass outputs between tasks (`taskValues`).

## When to use what

| Step | Task type |
|------|-----------|
| PySpark ETL | **Notebook** or **Pipeline** |
| Pure SQL transform | **SQL** |
| Refresh exec dashboard | **Dashboard** |
| Medallion in LDP | **Pipeline** |

## Exam traps

- **SQL task** without SQL warehouse configured.
- Duplicate **task_key** names in one job.
- Dashboard refresh task running **before** upstream table job completes (missing dependency).
