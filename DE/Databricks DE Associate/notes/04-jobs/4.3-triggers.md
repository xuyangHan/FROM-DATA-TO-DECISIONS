# 4.3 Job schedules & triggers

## Exam objective

> Implement job schedules using Lakeflow Jobs with an understanding of trigger types (scheduled, file arrival, and table update).

## Must know

- **Scheduled (cron)**: time-based; quartz cron in UTC typically.
- **File arrival**: trigger when new files land in cloud path (paired with ingestion pattern).
- **Table update**: trigger when Delta table receives updates (data-driven downstream).
- **Continuous** (pipelines): always-on for streaming/LDP — distinct from batch cron.
- Paused schedules vs disabled jobs.

## When to use what

| Trigger | Use when |
|---------|----------|
| **Scheduled** | Predictable batch window (nightly ETL) |
| **File arrival** | Pipeline should run when drops land in storage |
| **Table update** | Downstream only when upstream UC table commits |

## Exam traps

- **Cron** timezone confusion (UTC vs local).
- **File arrival** on path with no new files — job never runs vs runs empty (know product behavior).
- Using **schedule** when question says “run only when table X updates” → **table update**.
