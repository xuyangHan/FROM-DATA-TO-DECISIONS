# 4.1 Lakeflow Jobs control flow

## Exam objective

> Implement control flows (retries and conditional tasks such as branching and looping) using Lakeflow Jobs for pipeline orchestration.

## Must know

- **Retries**: per-task `max_retries`, `min_retry_interval_millis` on transient failures.
- **Timeout**: `timeout_seconds` at task or job level.
- **Condition tasks**: if/else branching on task outcome or parameters (`{{tasks.xxx.result_state}}`).
- **For each** / parameterized runs: loop over values (e.g. date list) — check current Jobs UI naming.
- **Depends on**: upstream success required unless `run_if` / dependency conditions configured.

## When to use what

| Need | Feature |
|------|---------|
| Flaky network to source | **Retries** on ingest task |
| Skip downstream on failure | **Condition** on `FAILED` vs `SUCCESS` |
| Process N regions | **For each** / parameter matrix |
| Long-running safety | **Timeout** |

## Exam traps

- Retries on **logic errors** (bad SQL) — wastes resources; fix code instead.
- **Condition** wired to wrong upstream task key.
- Infinite loop via **cyclic dependencies** in DAG (not allowed).
