# 3.1 Bronze → Silver cleaning

## Exam objective

> Implement data cleaning by reading bronze tables with PySpark/SQL, cleaning nulls, standardizing data types, and writing to new silver tables.

## Must know

- **Medallion**: bronze (raw) → silver (cleaned/conformed) → gold (aggregates/BI).
- Never overwrite bronze in place for auditability — write **new silver** table.
- **Nulls**: `dropna`, `fillna`, or `filter(col.isNotNull())`.
- **Types**: `cast`, `to_date`, `to_timestamp` for standardization.
- **Strings**: `trim`, `lower`, regex replace for consistency.
- UC: `main.silver.*` governed tables.

## When to use what

| Issue | Fix |
|-------|-----|
| Bad rows | Filter out or quarantine table (3.7) |
| Missing dims | `fillna` or leave null for left join later |
| Wrong type | `cast` with invalid rows → null + filter |

## PySpark / API essentials

```python
from pyspark.sql.functions import col, trim, lower, to_timestamp

bronze = spark.table("main.bronze.orders")

silver = (
  bronze
  .filter(col("order_id").isNotNull())
  .withColumn("email", lower(trim(col("email"))))
  .withColumn("order_ts", to_timestamp(col("order_ts_str"), "yyyy-MM-dd HH:mm:ss"))
  .drop("order_ts_str")
  .dropDuplicates(["order_id"])
)

silver.write.format("delta").mode("overwrite").saveAsTable("main.silver.orders")
```

## SQL (exam)

```sql
CREATE OR REPLACE TABLE main.silver.orders AS
SELECT order_id, LOWER(TRIM(email)) AS email,
       TO_TIMESTAMP(order_ts_str) AS order_ts
FROM main.bronze.orders
WHERE order_id IS NOT NULL;
```

## Exam traps

- **In-place** bronze overwrite (loses raw history).
- `cast` without handling errors → nulls silently or failed job.
- Silver table **not registered** in UC (`save` path only).
