# 3.3 Column & row manipulation

## Exam objective

> Manipulate columns, rows, and table structures by adding, dropping, splitting, renaming column names, applying filters, and exploding arrays.

## Must know

- **withColumn** / **select** — add or replace columns.
- **drop** — remove columns.
- **withColumnRenamed** / `rename` — rename.
- **filter** / **where** — row predicates (equivalent).
- **split** — string → array; often paired with **element_at** or **explode**.
- **explode** / **posexplode** — array/struct → one row per element.
- Exploding **null** or empty arrays → zero rows for that record.

## When to use what

| Task | API |
|------|-----|
| Remove PII column | `drop("ssn")` |
| Parse `full_name` | `split(col("full_name"), " ")` |
| Tags array → rows | `explode(col("tags"))` |
| Keep good rows only | `filter(col("status") == "ok")` |

## PySpark / API essentials

```python
from pyspark.sql.functions import col, split, explode, lit

df = spark.table("main.bronze.events")

clean = (
  df
  .filter(col("event_time").isNotNull())
  .withColumnRenamed("userId", "user_id")
  .withColumn("year", split(col("event_date"), "-").getItem(0))
  .withColumn("tag", explode(col("tags")))  # tags: array<string>
  .drop("raw_payload")
)

# split without explode
df2 = df.withColumn("parts", split(col("path"), "/"))

clean.write.format("delta").mode("overwrite").saveAsTable("main.silver.events_flat")
```

## SQL (exam)

```sql
SELECT user_id, EXPLODE(tags) AS tag
FROM main.bronze.events
WHERE event_time IS NOT NULL;
```

## Exam traps

- **explode** on null array removes the row entirely (use `explode_outer` to keep nulls).
- **select*** after explode without understanding duplicate grain.
- Renaming with **withColumn** same name — replaces column (intended) but easy to confuse with add.
