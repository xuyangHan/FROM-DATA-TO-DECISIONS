# 3.6 Gold layer objects

## Exam objective

> Understand the difference between, and how to build, Gold layer objects such as materialized views, views, streaming tables, and tables for BI and analytics teams in Unity Catalog.

## Must know

- **Table (managed Delta)**: physical storage; full CRUD; best for curated datasets.
- **View**: logical SQL definition; always reflects current underlying data; no storage cost for data itself.
- **Materialized view (MV)**: precomputed snapshot; refreshed on schedule or trigger; faster BI queries.
- **Streaming table**: Delta table fed by continuous/batch pipeline (LDP); for near-real-time gold.
- Gold serves **BI / analytics** — stable schema, documented metrics, UC permissions.

## When to use what

| Object | Use when |
|--------|----------|
| **Gold table** | Canonical metrics dataset; heavy transforms materialized |
| **View** | Thin rename/join layer; always fresh; low storage |
| **Materialized view** | Expensive aggregations queried repeatedly by BI |
| **Streaming table** | Dashboards need low-latency updates from stream |

## PySpark / API essentials

```python
# Gold managed table (PySpark)
(spark.table("main.silver.orders")
  .groupBy("order_date")
  .sum("amount")
  .withColumnRenamed("sum(amount)", "revenue")
  .write.format("delta").mode("overwrite")
  .saveAsTable("main.gold.daily_revenue"))

# Temp view for exploration only (not UC gold for production BI)
df.createOrReplaceTempView("tmp_revenue")
```

## SQL (exam)

```sql
CREATE OR REPLACE VIEW main.gold.orders_enriched AS
SELECT o.*, c.segment
FROM main.silver.orders o
LEFT JOIN main.silver.customers c ON o.customer_id = c.customer_id;

CREATE MATERIALIZED VIEW main.gold.daily_revenue_mv AS
SELECT order_date, SUM(amount) AS revenue
FROM main.silver.orders
GROUP BY order_date;

-- Refresh MV (syntax may vary by platform version)
REFRESH MATERIALIZED VIEW main.gold.daily_revenue_mv;
```

## UI / product surfaces

- **SQL editor** / **Catalog** → create view, MV, streaming table (LDP UI).

## Exam traps

- **View** when BI needs pre-aggregated performance (should be MV or table).
- **MV** when data must be real-time per row (view or streaming table).
- **Temp view** presented as enterprise gold asset (not UC-governed).
