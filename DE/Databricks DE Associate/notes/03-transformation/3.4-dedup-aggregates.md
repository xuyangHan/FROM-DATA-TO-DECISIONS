# 3.4 Deduplication & aggregates

## Exam objective

> Perform data deduplication operations and aggregate operations on DataFrames, such as count, approximate count distinct, and mean, summary.

## Must know

- **dropDuplicates(subset=[...])** — keep first row per key subset; full row duplicate needs all columns match.
- **groupBy + agg** — `count`, `avg`, `sum`, custom aggregations.
- **approx_count_distinct** — faster approximate distinct on high cardinality (HyperLogLog).
- **summary()** / **describe()** — quick stats (count, mean, stddev, min, max).
- **count** on full DF is expensive (full scan); **countDistinct** even more so — use approx when acceptable.

## When to use what

| Goal | Method |
|------|--------|
| Remove duplicate keys | `dropDuplicates(["id"])` |
| Exact distinct count (small/medium) | `countDistinct("user_id")` |
| Billion-row cardinality estimate | `approx_count_distinct("user_id")` |
| Profile a column | `df.select("amount").summary()` |

## PySpark / API essentials

```python
from pyspark.sql.functions import col, count, avg, approx_count_distinct

df = spark.table("main.bronze.clickstream")

deduped = df.dropDuplicates(["event_id", "event_time"])

metrics = (
  deduped.groupBy("country")
  .agg(
    count("*").alias("events"),
    approx_count_distinct("user_id").alias("users_approx"),
    avg("session_seconds").alias("avg_session"),
  )
)

deduped.groupBy("country").count().show()  # simple aggregate
deduped.select("session_seconds").summary().show()

metrics.write.format("delta").mode("overwrite").saveAsTable("main.silver.country_metrics")
```

## SQL (exam)

```sql
SELECT country,
       COUNT(*) AS events,
       APPROX_COUNT_DISTINCT(user_id) AS users_approx,
       AVG(session_seconds) AS avg_session
FROM main.silver.clickstream
GROUP BY country;
```

## Exam traps

- **dropDuplicates()** without subset when business key is only `id` (duplicates on other cols remain).
- Using **countDistinct** on huge columns when question allows **approx_count_distinct**.
- **mean** on nulls — `avg` ignores nulls; know `count` vs `count(*)` difference in SQL.
