# 3.2 Joins & unions

## Exam objective

> Combine DataFrames with operations such as Inner join, left join, broadcast join, multiple keys, cross join, union, and union all.

## Must know

- **Inner join**: only matching keys in both sides.
- **Left join**: all left rows; nulls where no right match.
- **Broadcast join**: small table replicated to all executors — avoids shuffle on large-small joins.
- **Multiple keys**: pass list of column names to `on`.
- **Cross join**: Cartesian product — expensive; rare in exams except “what happens”.
- **Union**: removes duplicates (distinct); **unionAll** / `unionByName(allowMissingColumns=True)` keeps all rows.
- Wrong join key → fan-out duplicates (row multiplication).

## When to use what

| Situation | Approach |
|-----------|----------|
| Large fact + small dimension | **Broadcast** small side (`broadcast(df_small)`) |
| Both tables large | Sort-merge shuffle join (default); tune partitions (3.5) |
| Need all left records | **Left** join |
| Stack same-schema datasets | **unionByName** |
| Columns differ slightly | `unionByName(..., allowMissingColumns=True)` |

## PySpark / API essentials

```python
orders = spark.table("main.silver.orders")
customers = spark.table("main.silver.customers")

# Inner join, single key
joined = orders.join(customers, on="customer_id", how="inner")

# Multiple keys
joined = orders.join(customers, on=["customer_id", "region"], how="left")

# Broadcast hint (small dimension < ~10MB default threshold)
from pyspark.sql.functions import broadcast
joined = orders.join(broadcast(customers), on="customer_id", how="inner")

# Cross join (avoid unless intentional)
crossed = orders.crossJoin(customers.limit(1))

# Union — prefer unionByName for column order safety
df_a = spark.table("main.silver.events_2024")
df_b = spark.table("main.silver.events_2025")
stacked = df_a.unionByName(df_b, allowMissingColumns=True)
```

## SQL (exam)

```sql
SELECT o.*, c.name
FROM main.silver.orders o
INNER JOIN main.silver.customers c
  ON o.customer_id = c.customer_id AND o.region = c.region;

-- Broadcast hint in SQL
SELECT /*+ BROADCAST(c) */ ...
FROM orders o JOIN customers c ON ...
```

## UI / product surfaces

- **Spark UI** → SQL/DataFrame tab — watch for **Exchange** (shuffle) vs **BroadcastExchange**.

## Exam traps

- **Left join** when question wants only matched rows (should be inner).
- **Union** vs **union all** — union dedupes; union all keeps duplicates.
- Forgetting **duplicate keys** on one side multiplies rows.
- **Cross join** by missing join condition in SQL.
