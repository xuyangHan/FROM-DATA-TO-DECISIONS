# 2.2 COPY INTO

## Exam objective

> Use the COPY INTO command to incrementally load files from cloud object storage (ADLS/S3/GCS) into Unity-Catalog-governed tables.

## Must know

- **COPY INTO** is SQL-first; loads files from a **cloud path** into a **Delta UC table**.
- **Incremental**: tracks files already loaded; only new files in path are ingested on subsequent runs.
- Supports **CSV, JSON, Parquet**, etc. via `FILEFORMAT`.
- Target must be a **Unity Catalog** table (three-level name).
- Credentials via **storage credential** / external location in UC.
- Best for **moderate file volume** batch loads — not millions of tiny files (see 2.6).

## When to use what

| vs Auto Loader | Choose COPY INTO when |
|----------------|----------------------|
| Simpler SQL ops team | Yes |
| Millions of files / notifications | No → Auto Loader |
| Strict scheduled batch | Yes |

## PySpark / API essentials

```python
# COPY INTO is SQL — run in SQL cell or spark.sql
spark.sql("""
  COPY INTO main.bronze.orders_raw
  FROM 's3://bucket/orders/'
  FILEFORMAT = PARQUET
  FORMAT_OPTIONS ('mergeSchema' = 'true')
  COPY_OPTIONS ('force' = 'false')
""")
```

## SQL (exam)

```sql
COPY INTO main.bronze.orders_raw
FROM 'abfss://container@account.dfs.core.windows.net/orders/'
FILEFORMAT = CSV
FORMAT_OPTIONS ('header' = 'true', 'inferSchema' = 'true')
COPY_OPTIONS ('force' = 'false');

-- force = true re-loads all files (exam: know it exists, use carefully)
```

## UI / product surfaces

- **SQL editor** + warehouse; or notebook `%sql` with UC enabled.

## Exam traps

- **force = true** on production without understanding re-ingest duplicates.
- Path not covered by **external location** → auth failure.
- Using COPY INTO for **continuous** file firehose (Auto Loader wins).
