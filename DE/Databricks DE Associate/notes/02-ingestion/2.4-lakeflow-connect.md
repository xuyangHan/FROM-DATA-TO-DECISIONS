# 2.4 Lakeflow Connect

## Exam objective

> Configure Lakeflow Connect to reliably ingest data from diverse enterprise sources into Unity-Catalog-governed tables.

## Must know

- **Connection**: credentials + source endpoint stored securely (UC/secrets).
- **Standard connectors**: you configure sync frequency and monitoring.
- **Managed connectors**: Databricks-operated ingestion with tighter SLA/ops.
- Destination: **UC Delta tables** (often bronze).
- Sync modes: full refresh vs incremental (connector-dependent).
- Lineage and permissions flow through Unity Catalog.

## When to use what

| Source | Path |
|--------|------|
| Supported SaaS (Salesforce, etc.) | **Managed** if available, else **standard** |
| Custom API not in Connect | JDBC/REST notebook (2.5) |
| Cloud files | Auto Loader / COPY INTO (not Connect) |

## PySpark / API essentials

```python
# Connect ingestion is UI/API-driven; consume result tables in PySpark
df = spark.table("main.bronze.salesforce_accounts")  # synced by Connect
```

## SQL (exam)

```sql
SELECT COUNT(*), MAX(_sync_timestamp) FROM main.bronze.salesforce_accounts;
```

## UI / product surfaces

- **Data engineering** → **Lakeflow Connect** → Create connection → Create sync → Target UC table.

## Exam traps

- Connect for **S3 file drops** (wrong tool — use Auto Loader).
- Storing credentials in notebook **plaintext** instead of connection/secret scope.
- Target table **not** in Unity Catalog when governance required.
