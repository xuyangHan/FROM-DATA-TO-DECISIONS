# 2.7 Semi-structured & unstructured data

## Exam objective

> Ingest semi-structured and unstructured data (for example, JSON and nested data) via Lakeflow Connect and other managed connectors into Unity-Catalog-governed Delta tables.

## Must know

- **Semi-structured data** has flexible or nested structure, such as JSON, XML, maps, arrays, structs, and SaaS event payloads.
- **Unstructured data** has no table-like schema, such as PDFs, images, audio, video, raw text documents, and model artifacts.
- The exam-safe landing pattern is still **bronze Unity Catalog Delta tables** for tabular/semi-structured rows and **Unity Catalog volumes** for raw unstructured files.
- **JSON files in cloud storage** are usually ingested with Auto Loader when files arrive repeatedly, at scale, or with schema drift.
- **SaaS/API nested data** should use Lakeflow Connect when a supported connector exists; otherwise use REST/JDBC notebooks scheduled with Lakeflow Jobs.
- **Bronze** should preserve source fidelity: raw JSON string, `VARIANT`, structs, arrays, metadata columns, rescued data, and source file info.
- **Silver** should parse, type, validate, flatten, explode, deduplicate, and standardize fields for analytics.
- **Schema inference** on sparse or inconsistent JSON can be risky. Prefer explicit schemas, schema hints, rescued data, or `VARIANT` depending on the use case.
- **`VARIANT`** stores flexible semi-structured data in Delta tables in Databricks Runtime 15.3 and above. It is useful when schema is unknown or changes often.
- **Structs, arrays, and maps** are native Spark/Delta complex types. They are best when you know the schema and want typed columns.
- **Frequently filtered fields** should usually become normal columns for performance, statistics, clustering, and simpler BI access.
- Use **`explode`** or **`explode_outer`** to flatten arrays into one row per element. `explode_outer` keeps rows when arrays are null.
- Use **`from_json`** to parse JSON strings into structs and **`parse_json`** to parse JSON strings into `VARIANT`.

## Data Type Choices

| Data shape | Best storage pattern | Use when | Watch out |
|------------|----------------------|----------|-----------|
| Stable JSON schema | Structs, arrays, maps in Delta | You know fields and types | Schema evolution is stricter |
| Unknown or changing JSON/XML | `VARIANT` plus extracted key columns | Schema changes often or has many fields | Variant has query/optimization limitations |
| Raw payload needed for replay | Raw JSON string or `VARIANT` in bronze | Need audit and reprocessing | Extract common fields to columns |
| Nested arrays of business entities | Struct/array in bronze, explode in silver | Need normalized analytics tables | Explode changes row grain |
| PDFs/images/audio/video | Unity Catalog volume + metadata Delta table | File governance and downstream ML/AI | Not a regular Delta table by itself |

Exam shortcut: **known schema = structs**; **unknown/frequently changing schema = variant or raw JSON + rescued data**; **files that are not rows = volumes**.

## Ingestion Decision Table

| Scenario | Ingest with | Refine with |
|----------|-------------|-------------|
| High-volume JSON files in S3/ADLS/GCS/Volumes | Auto Loader | Schema hints, rescued data, silver parsing |
| Small scheduled JSON file drops owned by SQL team | `COPY INTO` | SQL/PySpark transforms |
| SaaS connector emits nested objects | Lakeflow Connect | Silver flattening and type cleanup |
| Custom REST API returns nested JSON | REST notebook/Python task + Lakeflow Job | `from_json`, `explode`, `select` |
| Kafka events have JSON payload strings | Standard connector / Structured Streaming | `from_json` or `parse_json` |
| XML files or strings | Auto Loader / `read_files`, then XML parsing | `from_xml` / `VARIANT` where supported |
| PDFs or images | UC volume | Metadata table with path, source, labels, status |

## Bronze To Silver Pattern

```
Bronze
  -> Preserve raw payload and source metadata
  -> Store malformed or unexpected data in rescued columns
  -> Keep arrays/structs nested if that mirrors source

Silver
  -> Parse JSON strings
  -> Cast important fields to stable types
  -> Extract frequently queried fields
  -> Explode arrays only when the row grain should change
  -> Filter invalid rows and enforce quality rules

Gold
  -> Analytics-friendly tables, views, materialized views
```

Do not over-flatten bronze. Keeping raw source shape makes replay and debugging easier.

## Common Functions And Concepts

| Function / concept | Purpose |
|--------------------|---------|
| `from_json(col, schema)` | Parse JSON string column into typed struct/array |
| `schema_of_json(json_string)` | Infer schema from a JSON sample |
| `parse_json(col)` | Parse JSON string into `VARIANT` |
| `variant_get` / `try_variant_get` | Extract and cast values from `VARIANT` |
| `schema_of_variant` / `schema_of_variant_agg` | Inspect variant structure |
| `explode(col)` | Turn array/map entries into rows; drops null/empty arrays |
| `explode_outer(col)` | Explode while preserving null rows |
| `get_json_object` | Extract a JSON path from a string as a string |
| `to_json(col)` | Convert struct/array/map/variant to JSON string |
| `select("struct.field")` | Select nested struct fields |
| `col("arr")[0]` | Access array element by index |
| `_rescued_data` | Store unexpected or malformed fields from Auto Loader |

## PySpark / API Essentials

```python
# Ingest nested JSON with Auto Loader into a bronze UC table.
(spark.readStream.format("cloudFiles")
  .option("cloudFiles.format", "json")
  .option("cloudFiles.schemaLocation", "/Volumes/main/landing/_schemas/events")
  .option("cloudFiles.inferColumnTypes", "true")
  .option("cloudFiles.schemaEvolutionMode", "addNewColumns")
  .option("rescuedDataColumn", "_rescued_data")
  .load("/Volumes/main/landing/events_json")
  .writeStream
  .option("checkpointLocation", "/Volumes/main/landing/_checkpoints/events_json")
  .option("mergeSchema", "true")
  .trigger(availableNow=True)
  .toTable("main.bronze.events_raw"))
```

```python
# Parse a JSON string payload into a typed struct.
from pyspark.sql import functions as F
from pyspark.sql.types import (
    StructType, StructField, StringType, DoubleType, ArrayType
)

payload_schema = StructType([
    StructField("event_id", StringType()),
    StructField("user_id", StringType()),
    StructField("amount", DoubleType()),
    StructField("tags", ArrayType(StringType())),
    StructField("attrs", StructType([
        StructField("color", StringType()),
        StructField("campaign", StringType()),
    ])),
])

bronze = spark.table("main.bronze.api_events_raw")

parsed = bronze.withColumn("payload_struct", F.from_json("payload", payload_schema))
```

```python
# Flatten selected fields and explode tags in silver.
silver = (
    parsed
    .select(
        F.col("payload_struct.event_id").alias("event_id"),
        F.col("payload_struct.user_id").alias("user_id"),
        F.col("payload_struct.amount").alias("amount"),
        F.col("payload_struct.attrs.color").alias("color"),
        F.explode_outer("payload_struct.tags").alias("tag"),
        F.col("_rescued_data")
    )
    .filter(F.col("event_id").isNotNull())
)

silver.write.mode("overwrite").saveAsTable("main.silver.events_flat")
```

```python
# Infer schema from a small sample only when appropriate.
from pyspark.sql.functions import schema_of_json, lit

sample_json = '{"event_id":"e1","amount":12.5,"tags":["promo"]}'
inferred_schema = spark.range(1).select(schema_of_json(lit(sample_json))).first()[0]
```

```python
# Unstructured files: store files in a UC volume and track metadata in a table.
files = (spark.read.format("binaryFile")
  .load("/Volumes/main/landing/documents"))

(files.select("path", "modificationTime", "length")
  .write.mode("append")
  .saveAsTable("main.bronze.document_file_index"))
```

## SQL (exam)

```sql
-- Query nested struct fields.
SELECT
  payload_struct.event_id,
  payload_struct.attrs.color
FROM main.bronze.api_events_parsed;
```

```sql
-- Explode an array into rows.
SELECT
  event_id,
  tag
FROM main.bronze.events_raw
LATERAL VIEW OUTER explode(tags) exploded AS tag;
```

```sql
-- Store flexible JSON as VARIANT.
CREATE OR REPLACE TABLE main.bronze.events_variant AS
SELECT
  event_id,
  parse_json(raw_payload) AS payload
FROM main.bronze.events_raw_strings;
```

```sql
-- Query fields from a VARIANT column and cast them.
SELECT
  payload:event_id::STRING AS event_id,
  payload:amount::DOUBLE AS amount,
  payload:attrs.color::STRING AS color
FROM main.bronze.events_variant;
```

```sql
-- Keep frequently filtered fields as normal columns.
CREATE OR REPLACE TABLE main.silver.events AS
SELECT
  payload:event_id::STRING AS event_id,
  payload:user_id::STRING AS user_id,
  payload:event_type::STRING AS event_type,
  payload AS raw_payload
FROM main.bronze.events_variant;
```

## Variant vs Struct vs JSON String

| Need | Choose |
|------|--------|
| Fast BI queries on known fields | Struct columns and extracted scalar columns |
| Flexible payload with changing schema | `VARIANT` |
| Lowest-friction raw landing | JSON string or `VARIANT` |
| Preserve exact original text | JSON string |
| Query nested dynamic fields | `VARIANT` with `:` paths or variant functions |
| Use fields for partitioning/clustering/stats | Extract them to normal columns |

Variant caveat: `VARIANT` is flexible, but it is not a magic replacement for modeling. Extract important fields for filtering, joining, grouping, clustering, and BI.

## UI / Product Surfaces

- **Catalog Explorer**: inspect nested schemas, table lineage, owners, tags, and permissions.
- **Notebook schema view / DataFrame display**: inspect structs, arrays, maps, and inferred types.
- **Auto Loader / pipeline event logs**: monitor schema evolution, rescued data, and malformed records.
- **Volumes**: govern raw unstructured files such as images, PDFs, audio, and video.
- **Lakeflow Connect / Add data**: ingest semi-structured SaaS or source records when a connector exists.

## Exam Traps

- Inferring schema from sparse or inconsistent JSON and trusting the result blindly.
- Parsing and flattening everything in bronze, which makes raw replay and debugging harder.
- Exploding arrays without noticing the row grain changed.
- Using `explode` when `explode_outer` is needed to preserve rows with null arrays.
- Dropping malformed or unexpected fields instead of using rescued data or raw payload retention.
- Treating unstructured files as Delta table rows without a file storage strategy. Use UC volumes plus metadata tables.
- Keeping important filter/join columns only inside JSON string or `VARIANT`, which hurts performance and usability.
- Assuming field names are case-insensitive in JSON/variant path access.
- Assuming all connector metadata column names are the same across Lakeflow Connect sources.
- Using `VARIANT` on runtimes or table formats where it is unsupported.
