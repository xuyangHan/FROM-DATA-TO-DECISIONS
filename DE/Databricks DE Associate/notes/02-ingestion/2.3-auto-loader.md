# 2.3 Auto Loader

## Exam objective

> Use Auto Loader with schema enforcement and schema evolution in batch modes (for example, directory listing or file notification) to land data into Unity-Catalog-governed tables.

## Must know

- Format: **`cloudFiles`** (Auto Loader) on `readStream`.
- **Directory listing**: scans path (good default); **file notification** (SQS/Event Grid) for huge scale.
- **schemaEvolution**: `rescue` column for unexpected fields; `mergeSchema` / `failOnNewColumns` modes.
- **schemaHints**: fix types when inference wrong.
- **Checkpoint** required; stores discovered files state.
- Can run in **triggered** mode (`Trigger.AvailableNow`) for batch-style micro-batches.

## When to use what

| Option | When |
|--------|------|
| Directory listing | Most workloads |
| File notification | Very high file arrival rate, cloud supported |
| schemaEvolution | JSON/CSV schema drift |
| failOnNewColumns | Strict production schema |

## PySpark / API essentials

```python
(spark.readStream.format("cloudFiles")
  .option("cloudFiles.format", "json")
  .option("cloudFiles.schemaLocation", "s3://bucket/_schemas/orders")
  .option("cloudFiles.inferColumnTypes", "true")
  .option("rescuedDataColumn", "_rescued")
  .schema("order_id STRING, amount DOUBLE")  # optional enforcement
  .load("s3://bucket/incoming/orders/")
  .writeStream
  .option("checkpointLocation", "s3://bucket/_checkpoints/orders")
  .option("mergeSchema", "true")
  .trigger(availableNow=True)
  .table("main.bronze.orders_raw"))
```

## Exam traps

- Forgetting **schemaLocation** when using evolution (metadata store).
- **read** instead of **readStream** (not Auto Loader).
- Same checkpoint path for **two different** streams (corruption).
