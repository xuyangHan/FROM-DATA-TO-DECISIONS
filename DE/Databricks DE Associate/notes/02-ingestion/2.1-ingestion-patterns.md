# 2.1 Ingestion patterns

## Exam objective

> Enable and detail data ingestion patterns, including batch, streaming, and incremental loading, and import data from sources such as local files, Lakeflow Connect standard connectors, and Lakeflow Connect managed connectors.

## Must know

- **Batch**: read all/new data in one job window; `read` + `write` / `saveAsTable`.
- **Streaming**: unbounded data; `readStream` + `writeStream`; **checkpoint** required for fault tolerance.
- **Incremental**: only new/changed records — Auto Loader, `COPY INTO`, CDC, or watermark on event time.
- **Bronze** layer: raw landing in UC Delta tables (see [3.1](../03-transformation/3.1-bronze-to-silver.md)).
- **Local files**: upload to Volumes or cloud staging then ingest.
- **Connect**: standard (you manage) vs managed (Databricks sync service).

## When to use what

| Pattern | When |
|---------|------|
| Batch | Daily/hourly snapshots; small files |
| Streaming | Near real-time events |
| Incremental | Large recurring drops; avoid full reload |
| Connect | SaaS/DB with connector |

## PySpark / API essentials

```python
# Batch
(spark.read.format("csv").option("header", True)
  .load("/Volumes/main/bronze/uploads/data.csv")
  .write.format("delta").mode("append").saveAsTable("main.bronze.uploads"))

# Streaming
(spark.readStream.table("main.bronze.kafka_events")  # or readStream from cloudFiles
  .writeStream
  .option("checkpointLocation", "/Volumes/main/bronze/checkpoints/kafka")
  .trigger(processingTime="1 minute")
  .table("main.silver.kafka_events"))
```

## Exam traps

- Streaming without **checkpointLocation** (cannot recover offsets).
- Full **overwrite** bronze on every run (loses incremental history).
- Confusing **batch readStream** (Auto Loader available now) with continuous streaming.
