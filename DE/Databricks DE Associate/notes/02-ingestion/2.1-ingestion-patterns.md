# 2.1 Ingestion patterns

## Exam objective

> Enable and detail data ingestion patterns, including batch, streaming, and incremental loading, and import data from sources such as local files, Lakeflow Connect standard connectors, and Lakeflow Connect managed connectors.

## Must know

- **Ingestion** is the process of landing source data into Databricks, usually into a **bronze** Unity Catalog Delta table.
- **Batch ingestion** processes a bounded dataset at a scheduled time or on demand. Common examples: daily snapshots, weekly CSV drops, one-time backfills.
- **Streaming ingestion** processes unbounded or continuously arriving data using Structured Streaming, Auto Loader, streaming tables, or connectors.
- **Incremental ingestion** processes only new or changed data since the last successful run. It avoids expensive full reloads.
- **Bronze layer** stores raw or lightly validated source data. Keep source fidelity here; heavy cleaning belongs in silver.
- **Unity Catalog target** is the exam-safe default: land data into `catalog.schema.table` so permissions, lineage, ownership, and discovery work.
- **Delta Lake target** is the default table format for reliable ingestion because it supports ACID transactions, schema enforcement, time travel, and batch plus streaming writes.
- **Checkpoint** is required for reliable Structured Streaming. It tracks offsets, committed micro-batches, and state so a stream can resume after failure.
- **`Trigger.AvailableNow`** is the modern pattern for incremental batch processing with Structured Streaming sources. It processes all currently available new records, then stops.
- **Watermark** tells Spark how long to wait for late-arriving event-time data in stateful streaming operations.
- **Local files** should be uploaded to a Unity Catalog volume or used to create/modify a Delta table from the Add data UI.
- **Lakeflow Connect managed connectors** are Databricks-managed ingestion pipelines for supported SaaS apps and databases, with source-specific auth, incremental ingestion, retries, schema evolution, and monitoring.
- **Lakeflow Connect standard connectors** give more customization and can be used through Structured Streaming or Lakeflow Spark Declarative Pipelines.
- **Community or partner connectors** are fallback choices when first-party managed or standard connectors do not cover the source.

## Ingestion mental model

```
Source system
  -> local file, cloud files, SaaS app, database, message bus, REST API
  -> ingestion method
       -> file upload, COPY INTO, Auto Loader, Lakeflow Connect, JDBC/REST notebook
  -> bronze Unity Catalog Delta table
  -> silver cleaning and validation
  -> gold serving tables, views, materialized views, or dashboards
```

For exam questions, first identify the **source type**, then the **arrival pattern**, then the **operational requirement**.

| Question clue | What to infer |
|---------------|---------------|
| "One-time" or "small local file" | Upload to volume or create table from file |
| "Daily snapshot" | Batch load |
| "Only new files since last run" | Incremental file ingestion |
| "Thousands or millions of files" | Auto Loader over manual file listing or simple SQL copy |
| "Near real-time events" | Streaming ingestion |
| "Supported SaaS or database connector" | Lakeflow Connect |
| "SQL team owns simple cloud-file load" | `COPY INTO` may be enough |
| "Custom REST API" | Notebook or Python task orchestrated by Lakeflow Jobs |

## Batch vs streaming vs incremental

| Pattern | Data shape | Typical tools | Best for | Watch out |
|---------|------------|---------------|----------|-----------|
| Batch | Bounded input | `spark.read`, SQL, file upload, `COPY INTO` | Snapshots, backfills, small or predictable drops | Full reloads can be slow and can duplicate data |
| Streaming | Unbounded input | `readStream`, Auto Loader, streaming tables, standard connectors | Logs, events, queues, frequent file arrival | Needs checkpoints and monitoring |
| Incremental batch | New/changed bounded input per run | Auto Loader with `AvailableNow`, `COPY INTO`, Lakeflow Connect, CDC, cursor columns | Scheduled loads that should skip old data | Requires state: checkpoint, loaded-file tracking, CDC log, or watermark/cursor |

Exam shortcut: **streaming API does not always mean always-on streaming**. Auto Loader with `Trigger.AvailableNow` uses Structured Streaming to do incremental batch ingestion, then stops.

## Source-to-tool decision table

| Source or requirement | Choose |
|-----------------------|--------|
| Local CSV/JSON/Excel sample file | Add data UI to create table, or upload to Unity Catalog volume |
| Non-tabular local files such as PDFs or images | Unity Catalog volume |
| Cloud object storage files, moderate volume, SQL-first team | `COPY INTO` |
| Cloud object storage files, high volume, schema drift, frequent arrival | Auto Loader |
| SaaS app with supported managed connector | Lakeflow Connect managed connector |
| Relational database with supported CDC connector | Lakeflow Connect managed database connector |
| Source supported by a standard connector and needs customization | Lakeflow Connect standard connector or Structured Streaming |
| Source has no connector and exposes JDBC | Notebook/Python task with JDBC, scheduled by Lakeflow Jobs |
| Source has no connector and exposes REST | Notebook/Python task with REST client, scheduled by Lakeflow Jobs |
| Need governance, lineage, and permissions | Land into Unity Catalog Delta tables |

## Lakeflow Connect connector types

| Connector type | What it means | Choose when |
|----------------|---------------|-------------|
| Managed connector | Fully managed Databricks connector backed by ingestion pipelines | Supported SaaS/database source and you want the lowest operational overhead |
| Standard connector | Connector with more control, often used through Structured Streaming or Lakeflow Spark Declarative Pipelines | You need customization or the source is not covered by managed connectors |
| Community connector | Open-source connector extending Lakeflow Connect | First-party managed/standard options do not support the source and beta/community support is acceptable |
| Partner connector | Third-party integration | The exam scenario explicitly says first-party Connect is unavailable or a partner tool is required |

## Incremental state patterns

| Pattern | State used to avoid reprocessing |
|---------|----------------------------------|
| Auto Loader | Checkpoint plus file discovery metadata |
| `COPY INTO` | Tracks files already loaded into the target table |
| Structured Streaming | Checkpoint offsets, commits, metadata, and state |
| CDC connector | Source change log, cursor, or connector-managed state |
| Query-based connector | Cursor or high-water mark column |
| Custom JDBC/REST job | Last processed timestamp, ID, version, or landing audit table |

## PySpark / API essentials

```python
# Batch load from a Unity Catalog volume into a bronze Delta table.
(spark.read.format("csv")
  .option("header", True)
  .option("inferSchema", True)
  .load("/Volumes/main/landing/uploads/orders.csv")
  .write
  .mode("append")
  .saveAsTable("main.bronze.orders_raw"))
```

```python
# Auto Loader as incremental batch: process all currently available new files, then stop.
(spark.readStream.format("cloudFiles")
  .option("cloudFiles.format", "json")
  .option("cloudFiles.schemaLocation", "/Volumes/main/landing/_schemas/orders")
  .load("/Volumes/main/landing/orders_json")
  .writeStream
  .option("checkpointLocation", "/Volumes/main/landing/_checkpoints/orders")
  .trigger(availableNow=True)
  .toTable("main.bronze.orders_raw"))
```

```python
# Continuous-style micro-batch stream from bronze to silver.
(spark.readStream.table("main.bronze.orders_raw")
  .withWatermark("event_time", "30 minutes")
  .writeStream
  .option("checkpointLocation", "/Volumes/main/landing/_checkpoints/orders_silver")
  .trigger(processingTime="1 minute")
  .toTable("main.silver.orders_clean"))
```

```python
# Simple custom REST pattern: fetch, convert to DataFrame, write to UC Delta.
# Schedule this notebook or Python task with Lakeflow Jobs.
import requests

response = requests.get(api_url, headers={"Authorization": f"Bearer {token}"})
response.raise_for_status()

records = response.json()["items"]
df = spark.createDataFrame(records)
df.write.mode("append").saveAsTable("main.bronze.api_events_raw")
```

## SQL (exam)

```sql
-- SQL-first incremental file ingestion from cloud object storage.
COPY INTO main.bronze.orders_raw
FROM 's3://company-landing/orders/'
FILEFORMAT = JSON
COPY_OPTIONS ('force' = 'false');
```

```sql
-- Confirm the bronze table is governed and discoverable through UC.
DESCRIBE DETAIL main.bronze.orders_raw;
DESCRIBE HISTORY main.bronze.orders_raw;
```

## UI / product surfaces

- **New > Data / Add data**: upload local files, create or modify a table, or upload files to a Unity Catalog volume.
- **Catalog Explorer**: browse target catalogs, schemas, tables, volumes, permissions, and lineage.
- **Lakeflow Connect / Data ingestion**: create managed ingestion pipelines for supported SaaS apps and databases.
- **Lakeflow Jobs**: schedule batch notebooks, custom JDBC/REST ingestion, SQL tasks, and ingestion pipelines.
- **Lakeflow Spark Declarative Pipelines**: define managed ETL pipelines, streaming tables, expectations, and refresh behavior.
- **Structured Streaming UI / query profile**: monitor running streams, batches, input rates, processing time, and failures.

## Exam traps

- Choosing a full reload when the question asks for new or changed data only.
- Writing bronze data to an unregistered path when Unity Catalog governance or lineage is required.
- Forgetting `checkpointLocation` for a streaming write.
- Reusing the same checkpoint path for two different streams.
- Confusing batch frequency with ingestion pattern: an hourly job can still be batch, incremental batch, or streaming depending on how it reads data.
- Choosing `COPY INTO` for a high-volume file firehose with schema drift. Auto Loader is usually the better fit.
- Choosing a custom REST/JDBC notebook when a managed Lakeflow Connect connector exists.
- Treating local file upload as a production ingestion strategy for recurring large feeds.
- Overwriting the bronze table every run without preserving source history or handling duplicates.
- Assuming `readStream` means low-latency continuous processing; `AvailableNow` is a triggered incremental pattern.
