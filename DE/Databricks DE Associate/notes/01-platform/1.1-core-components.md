# 1.1 Platform core components

## Exam objective

> Understand the core components of the Databricks Data Intelligence Platform, such as its architecture, Delta Lake, and Unity Catalog.

## Must know

- **Lakehouse**: open storage (cloud object store) + ACID tables (Delta) + governance (Unity Catalog) + compute.
- **Workspace**: notebooks, jobs, repos; lives in control plane; data in cloud storage via UC.
- **Account vs workspace**: account-level identity/billing; workspace is where engineers work.
- **Delta Lake**: ACID transactions, time travel (`VERSION AS OF`), `MERGE`, schema enforcement, `OPTIMIZE` / `VACUUM`.
- **Unity Catalog hierarchy**: metastore → **catalog** → **schema** → tables/views/volumes/functions.
- **Volumes**: UC-governed files (managed/external) — preferred over legacy DBFS for new work.
- **Lineage**: UC tracks read/write relationships across tables.

## When to use what

| Need | Component |
|------|-----------|
| Governed tables & ACLs | **Unity Catalog** |
| Reliable incremental ETL storage | **Delta** tables |
| Ad hoc files / uploads | **Volumes** or cloud paths + register |
| Historical debugging | Delta **time travel** |
| Compaction / layout | **OPTIMIZE** (+ Liquid clustering — see 6.4) |

```
Cloud storage (S3/ADLS/GCS)
        ↓
   Delta files (Parquet + transaction log)
        ↓
Unity Catalog registration (catalog.schema.table)
        ↓
Compute (cluster / warehouse / job) reads & writes
```

## PySpark / API essentials

```python
# Read governed table
df = spark.table("main.bronze.events")

# Time travel
spark.sql("SELECT * FROM main.bronze.events VERSION AS OF 10").show()

# Write managed Delta table in UC
df.write.format("delta").mode("overwrite").saveAsTable("main.silver.events")

# OPTIMIZE (maintenance)
spark.sql("OPTIMIZE main.silver.events")
```

## SQL (exam)

```sql
DESCRIBE EXTENDED main.silver.events;
SELECT * FROM main.bronze.events TIMESTAMP AS OF '2025-01-01';
```

## UI / product surfaces

- **Catalog Explorer** — browse catalogs, lineage, permissions.
- **Storage** / **Volumes** — file access under UC.

## Exam traps

- **Data lake** (files only) vs **lakehouse** (Delta + governance).
- Storing managed table data **outside** UC metastore cloud root without understanding external tables.
- **DBFS** root as long-term production store (legacy; UC Volumes preferred).
