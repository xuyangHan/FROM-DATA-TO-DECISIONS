# 6.2 Jobs UI pipeline health

## Exam objective

> Use the Lakeflow Jobs UI to monitor pipeline health by interpreting job statuses, viewing DAG-based task graphs to spot upstream blockers, and tracking pipeline run times and failure rates.

## Must know

- **Statuses**: Pending, Running, Success, Failed, Skipped, Cancelled, Timeout.
- **DAG view**: upstream failure blocks downstream (often Skipped).
- **Critical path**: longest chain determines job end time.
- **Failure rate**: failed runs / total runs in window.
- **Repair run**: rerun failed tasks only (when supported).

## When to use what

| Observation | Action |
|-------------|--------|
| Downstream Skipped | Fix **first failed** upstream task |
| One task always slow | Optimize or resize cluster (3.5, 6.3) |
| High failure rate after deploy | Roll back bundle / notebook change |

## UI / product surfaces

- Job run → **Graph** view with task colors; **Logs** per task; **Metrics**.

## Exam traps

- Fixing **downstream** task when **upstream extract** failed.
- **Skipped** interpreted as success.
- Ignoring **cluster not acquired** (infra) vs code failure.

## Hands-on checklist

- [ ] Intentionally fail an upstream task; observe downstream skipped in DAG.
- [ ] Use repair run on failed task only.
- [ ] Export mental SLA: total job duration vs per-task duration.
