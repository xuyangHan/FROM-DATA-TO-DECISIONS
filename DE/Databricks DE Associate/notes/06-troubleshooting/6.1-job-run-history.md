# 6.1 Job run history trends

## Exam objective

> Identify trends in job performance using the Lakeflow Jobs run history view to compare current execution times against historical baselines.

## Must know

- **Run history**: list of past runs with status, duration, trigger type.
- Compare **p50/p95 duration** over last N runs vs today’s run.
- Regressions: new library, data volume spike, cluster resize, upstream delay.
- **Run ID** links to Spark UI for deep dive (6.3).
- Filter failed vs successful runs for pattern spotting.

## When to use what

| Signal | Next step |
|--------|-----------|
| Duration +30% vs 7-day avg | Open slow run Spark UI |
| Failures clustered | Check upstream task / recent deploy |
| Success but slow | Data volume or skew |
