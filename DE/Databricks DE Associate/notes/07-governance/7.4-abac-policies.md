# 7.4 Unity Catalog ABAC policies

## Exam objective

> Understand Unity Catalog ABAC policies to centrally control row-level filtering and column masking for sensitive data.

## Must know

- **ABAC** (attribute-based access control): policies on **tags/attributes** (e.g. `pii=true`) not per-table manual setup.
- **Policy types**: row filter policies, column mask policies applied to matching assets.
- **Central governance**: one policy → many tables with same classification tag.
- Contrasts with **7.3** per-table masks/filters.
- **Classifications** / **tags** on columns/tables drive policy matches.

## When to use what

| Scale | Approach |
|-------|----------|
| Few tables | Per-table mask/filter (7.3) |
| Enterprise wide PII tagging | **ABAC policies** |
| New tables auto-tagged | Policies apply without new DDL |

## UI / product surfaces

- **Catalog** → **Policies** (ABAC) → create row/column policy → define tag conditions.

## Exam traps

- ABAC vs **manual GRANT** — policies don't replace need for USE CATALOG/SELECT.
- Forgetting to **tag** assets (policy never applies).
- Per-table mask when question asks for **central** control across hundreds of tables → ABAC.
