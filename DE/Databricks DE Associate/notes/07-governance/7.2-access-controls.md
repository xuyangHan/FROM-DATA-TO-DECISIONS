# 7.2 GRANT / REVOKE / DENY

## Exam objective

> Configure access controls using the UI and SQL by applying GRANT, REVOKE, and DENY privileges to principals (users, groups, and service principals) at appropriate levels of the security hierarchy.

## Must know

- **Hierarchy**: metastore → catalog → schema → table/view/volume/column (column privileges where supported).
- **Principals**: users, **groups**, **service principals**.
- **Privileges**: `SELECT`, `MODIFY`, `CREATE TABLE`, `USE CATALOG`, `USE SCHEMA`, `ALL PRIVILEGES`, etc.
- **GRANT** gives access; **REVOKE** removes; **DENY** explicit block (overrides grant in UC model).
- **Ownership**: owner can grant; metastore admin for top-level.
- Least privilege: grant at **lowest** level needed.

## When to use what

| Need | Grant at |
|------|----------|
| BI read one table | Table `SELECT` to group |
| ETL write silver | Schema `MODIFY` + `USE SCHEMA` to SP |
| New team catalog | Catalog `USE CATALOG` + schema creates |

## PySpark / API essentials

```python
# ACLs are SQL/admin — notebooks use spark.sql by metastore admin
spark.sql("GRANT SELECT ON TABLE main.gold.revenue TO `analysts@company.com`")
```

## SQL (exam)

```sql
GRANT USE CATALOG ON CATALOG main TO analysts;
GRANT USE SCHEMA ON SCHEMA main.gold TO analysts;
GRANT SELECT ON TABLE main.gold.revenue TO analysts;

REVOKE SELECT ON TABLE main.gold.revenue FROM analysts;

DENY SELECT ON TABLE main.gold.revenue TO contractors;

SHOW GRANTS ON TABLE main.gold.revenue;
```

## UI / product surfaces

- **Catalog** → Permissions tab → Add principal → privilege checkboxes.

## Exam traps

- Granting **SELECT** without **USE CATALOG/SCHEMA** (user cannot reach table).
- **DENY** breaking inherited group access (intentional but exam scenario).
- Using **all-purpose personal** token instead of **service principal** for jobs.
