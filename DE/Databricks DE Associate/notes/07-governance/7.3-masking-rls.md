# 7.3 Column masking & row filters

## Exam objective

> Understand column-level masking and row-level security to restrict data visibility based on user groups.

## Must know

- **Column mask**: function redacts/hashes column values for users without full access (e.g. show last-4 SSN).
- **Row filter**: predicate limits visible rows by user/group (e.g. `region = user_region()`).
- Applied on **Unity Catalog** tables; enforced for SQL/Spark reads through UC.
- **Dynamic views** (legacy pattern): views with `CURRENT_USER()` — prefer UC native filters/masks.
- Masks/filters assigned to **groups**, not individual notebooks.

## When to use what

| Requirement | Mechanism |
|-------------|-----------|
| Hide PII column | **Column mask** |
| Regional sales see own region only | **Row filter** |
| Central policy across many tables | ABAC (7.4) |

## PySpark / API essentials

```python
# Consumers read table normally — UC enforces mask/filter

```

## SQL (exam)

```sql
-- Admin defines (simplified — functions created separately)
CREATE FUNCTION main.mask_email(email STRING)
RETURN CASE WHEN IS_MEMBER('pii_readers') THEN email ELSE '***' END;

ALTER TABLE main.gold.customers ALTER COLUMN email SET MASK main.mask_email;

CREATE FUNCTION main.filter_region(region STRING)
RETURN IF(IS_MEMBER('admins'), true, region = CURRENT_USER_REGION());

ALTER TABLE main.gold.customers SET ROW FILTER main.filter_region ON (region);
```

## Exam traps

- Masking in **BI tool only** (not UC-enforced).
- Row filter forgetting **admins** exception group.
- Applying mask to **external** table without UC support/version awareness.
