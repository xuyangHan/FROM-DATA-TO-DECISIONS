# 7.1 Managed vs external tables

## Exam objective

> Differentiate between managed and external tables in Unity Catalog and perform basic operations (create, modify, delete, and convert between managed and external tables) on them.

## Must know

- **Managed table**: UC/Delta controls **storage location**; `DROP TABLE` removes data (by default).
- **External table**: data files live at explicit **`LOCATION`**; `DROP TABLE` drops metadata only (data remains).
- **Unity Catalog** three-level names required.
- **Convert**: implications for storage path, backups, and permissions.
- **Volumes** vs tables for unstructured files.

## When to use what

| Need | Table type |
|------|------------|
| Standard analytics tables | **Managed** |
| Share data already in S3 path | **External** |
| Compliance: must not delete files on drop | **External** |

## PySpark / API essentials

```python
# Managed
spark.sql("""
  CREATE TABLE main.silver.orders (id STRING, amount DOUBLE)
  USING DELTA
""")
df.write.format("delta").mode("overwrite").saveAsTable("main.silver.orders")

# External
spark.sql("""
  CREATE TABLE main.bronze.external_orders
  USING DELTA
  LOCATION 's3://bucket/bronze/orders/'
""")
```

## SQL (exam)

```sql
CREATE TABLE main.silver.orders_managed (id STRING)
USING DELTA;

CREATE TABLE main.bronze.orders_external (id STRING)
USING DELTA
LOCATION 'abfss://container@account.dfs.core.windows.net/orders/';

DROP TABLE main.silver.orders_managed;   -- deletes data
DROP TABLE main.bronze.orders_external;  -- metadata only
```

## UI / product surfaces

- **Catalog Explorer** → Create table → Managed vs External; **LOCATION** field.

## Exam traps

- **DROP external** assuming data deleted (compliance risk).
- External table without **storage credential** / external location access.
- Writing managed table to random path **outside** metastore root without intent.
