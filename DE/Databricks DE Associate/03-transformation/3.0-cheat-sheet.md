# Databricks Data Engineer Associate: Python, SQL, and Spark Cheat Sheet

Use this as a compact review map for the DE Associate exam. The exam likes syntax, return types, table behavior, and "what triggers execution?" questions.

## 1. Spark Session, Reading, and Writing

Create a Spark session:

```python
from pyspark.sql import SparkSession

spark = SparkSession.builder \
    .appName("MyApp") \
    .getOrCreate()
```

Read common file formats:

```python
df = spark.read.csv("/path/file.csv", header=True, inferSchema=True)
df = spark.read.json("/path/file.json")
df = spark.read.parquet("/path/file.parquet")
df = spark.read.format("delta").load("/path")
```

Write common outputs:

```python
df.write.mode("overwrite").parquet("/output")
df.write.format("delta").save("/output")
df.write.saveAsTable("sales")  # managed table by default
```

## 2. DataFrame Basics

Core transformations:

```python
from pyspark.sql.functions import col, lit

df.select("name", "age")
df.filter(df.age > 18)
df.filter(col("age") > 18)
df.where("age > 18")

df.withColumn("age_plus_1", col("age") + 1)
df.withColumn("country", lit("Canada"))
df.withColumnRenamed("old", "new")
df.drop("column_name")

df.select("country").distinct()
df.dropDuplicates()
df.dropDuplicates(["email"])

df.orderBy("age")
df.orderBy(col("age").asc())
df.orderBy(col("age").desc())
```

Classic valid vs invalid traps:

```python
col("age")                 # correct
col["age"]                 # wrong

df.withColumn("age", lit(5))  # correct
df.age = 5                    # wrong

when(col("x") > 5, "high")    # correct
if col("x") > 5:              # wrong for column logic
    ...
```

Use bitwise operators for column conditions:

```python
(col("age") > 18) & (col("country") == "CA")  # AND
(col("age") > 18) | (col("vip") == True)      # OR
~col("deleted")                               # NOT
```

Do not use Python `and` / `or` with Spark column expressions.

## 4. Type Conversion and Datetime

Cast columns:

```python
df.withColumn("age", col("age").cast("int"))
df.withColumn("salary", col("salary").cast("double"))
df.withColumn("created_ts", col("created_at").cast("timestamp"))
```

Parse and format dates/timestamps:

```python
df.withColumn("date", to_date(col("date_str"), "yyyy-MM-dd"))
df.withColumn("ts", to_timestamp(col("ts_str"), "yyyy-MM-dd HH:mm:ss"))
df.withColumn("formatted", date_format(col("ts"), "yyyy/MM/dd"))
```

Extract or calculate date parts:

```python
year(col("ts"))
month(col("ts"))
dayofmonth(col("ts"))
hour(col("ts"))

current_timestamp()
current_date()
datediff(col("end"), col("start"))  # returns days
date_add(col("date"), 7)
date_sub(col("date"), 7)
```

Datetime format tokens:

```text
yyyy -> year
MM   -> month
dd   -> day
HH   -> hour, 24-hour clock
mm   -> minute
ss   -> second
```

Return-type trap:

```text
to_date()      -> date
to_timestamp() -> timestamp
date_format()  -> string
```

## 5. Null Handling and Conditional Logic

Check, fill, drop, and replace nulls:

```python
col("name").isNull()
col("name").isNotNull()

df.na.fill(0)
df.na.fill({"age": 0})
df.na.drop()

coalesce(col("nickname"), col("name"))  # first non-null value
df.na.replace("UNKNOWN", None)
df.fillna(0)                            # alias for na.fill()
df.replace("M", "Male")                 # DataFrame-level replacement
```

Use `when` / `otherwise` for conditional columns:

```python
df.withColumn(
    "status",
    when(col("score") >= 60, "PASS").otherwise("FAIL")
)

df.withColumn(
    "grade",
    when(col("score") >= 90, "A")
    .when(col("score") >= 80, "B")
    .otherwise("C")
)
```

## 6. String Functions

Common string transforms:

```python
upper(col("name"))
lower(col("name"))
trim(col("name"))
concat(col("first"), col("last"))
concat_ws("-", col("year"), col("month"))
regexp_replace(col("phone"), "-", "")
split(col("email"), "@")          # returns array
col("email").contains("@gmail.com")
substring(col("id"), 1, 3)        # starts at 1, not 0
```

More string functions that can appear:

```python
translate(col("phone"), "-", "")          # character-by-character replacement
regexp_extract(col("email"), r"@(.*)", 1) # extract regex group
initcap(col("name"))                      # john smith -> John Smith
length(col("name"))
instr(col("email"), "@")
lpad(col("id"), 5, "0")                   # 12 -> 00012
rpad(col("id"), 5, "0")
substring_index(col("email"), "@", 1)     # text before @
col("email").startswith("admin")
col("email").endswith(".com")
col("email").like("%gmail%")
col("country").isin("CA", "US", "UK")     # SQL IN equivalent
col("age").between(18, 65)
```

Important comparison:

```text
translate()      -> character substitution
regexp_replace() -> regex pattern replacement
```

## 8. Arrays, Maps, Structs, and JSON

```python
array_contains(col("tags"), "spark")

```

Map functions:

```python

```

Struct and JSON functions:

```python

    StructField("name", StringType()),

to_json(col("struct_col"))
```


## 9. Aggregation, Grouping, Pivoting, and Set Operations

Basic aggregation:

```python
df.groupBy("department").count()

df.groupBy("department").agg(
    avg("salary").alias("avg_salary"),
    max("salary").alias("max_salary")
)
```

Common aggregate functions:

```python
count("*")
countDistinct("user_id")
approx_count_distinct("user_id")
avg("salary")
sum("sales")
max("salary")
min("salary")
first("salary")
last("salary")
collect_list("employee")  # keeps duplicates
collect_set("employee")   # removes duplicates
```

Pivot and reshape:

```python
df.groupBy("dept").pivot("gender").count()
```

`stack()` converts columns into rows and is often used through SQL or `selectExpr`.

Set-like DataFrame operations:

```python
df1.union(df2)          # requires same schema order
df1.unionByName(df2)    # matches columns by name
df1.intersect(df2)
df1.exceptAll(df2)
df.sample(0.1)          # 10 percent sample
df.limit(10)
```

## 10. Joins and Broadcast

Common joins:

```python
df1.join(df2, on="id", how="inner")
df1.join(df2, on="id", how="left")
```

Broadcast join:

```python
from pyspark.sql.functions import broadcast

df1.join(broadcast(df2), "id")
```

Exam model:

```text
Broadcast join -> small table sent to executors
Benefit        -> avoids shuffle on the large table
Risk           -> broadcasting a large table can cause memory issues
```

## 11. Window Functions

Define a window:

```python
from pyspark.sql.window import Window
from pyspark.sql.functions import row_number

windowSpec = Window.partitionBy("dept").orderBy(col("salary").desc())

df.withColumn("rank", row_number().over(windowSpec))
```

Common window functions:

```python
row_number().over(windowSpec)
rank().over(windowSpec)        # can skip numbers
dense_rank().over(windowSpec)  # no gaps
lag("salary", 1)               # previous row
lead("salary", 1)              # next row
```

## 12. Transformations, Actions, Shuffle, and Caching

Transformations are lazy:

```python
select()
filter()
join()
groupBy()
withColumn()
```

Actions trigger execution:

```python
show()
collect()
count()
take()
first()
write()
```

Narrow transformations do not require a shuffle:

```python
select()
filter()
withColumn()
```

Wide transformations usually require a shuffle:

```python
groupBy()
join()
distinct()
orderBy()
```

Shuffle is expensive because it involves network I/O, possible disk spill, and serialization.

Cache and persist:

```python
df.cache()

from pyspark import StorageLevel
df.persist(StorageLevel.MEMORY_AND_DISK)
```

Exam model:

```text
cache()   -> persist using the default storage level
persist() -> lets you choose the storage level
```

## 13. Driver, Executors, and OOM Patterns

Driver responsibilities:

```text
query planning
DAG generation
task scheduling
collecting results for driver-side actions
```

Executor responsibilities:

```text
actual computation
storing partitions
executing tasks
```

Driver OOM is commonly caused by:

```python
collect()
toPandas()
large broadcast table
```

Executor OOM is commonly caused by:

```text
huge partitions
skewed joins
insufficient executor memory
```

## 14. Partitioning and Performance

Partition data when large datasets are frequently filtered by the partition column:

```python
df.write.partitionBy("country")
```

Change DataFrame partitions:

```python
df.repartition(10)  # causes shuffle
df.coalesce(2)      # reduces partitions with minimal shuffle
```

Exam comparison:

```text
repartition() -> can increase or decrease partitions, full shuffle
coalesce()    -> usually reduces partitions, less shuffle
```

## 15. Delta Lake Essentials

Delta Lake is a storage layer that provides ACID transactions, schema enforcement, and time travel.

Create a Delta table:

```sql
CREATE TABLE sales (
    id INT,
    amount DOUBLE
)
USING DELTA;
```

Convert Parquet to Delta:

```python
from delta.tables import DeltaTable

DeltaTable.convertToDelta(
    spark,
    "parquet.`/path`"
)
```

Time travel:

```sql
SELECT * FROM sales VERSION AS OF 3;
SELECT * FROM sales TIMESTAMP AS OF '2025-01-01';
```

Modify Delta tables:

```sql
DELETE FROM sales WHERE amount < 0;

UPDATE sales
SET amount = amount * 1.1
WHERE country = 'CA';
```

Merge / upsert:

```sql
MERGE INTO target t
USING source s
ON t.id = s.id
WHEN MATCHED THEN
  UPDATE SET *
WHEN NOT MATCHED THEN
  INSERT *;
```

Clean and optimize:

```sql
VACUUM sales RETAIN 168 HOURS;
OPTIMIZE sales;
OPTIMIZE sales ZORDER BY (customer_id);
```

Exam model:

```text
VACUUM  -> removes old unused data files
OPTIMIZE -> compacts small files
ZORDER  -> improves data skipping for selected columns
```

## 16. Managed vs External Tables

Managed table:

```sql
CREATE TABLE sales (...);
```

```text
Databricks manages metadata and underlying files.
Dropping the table deletes the data.
```

External table:

```sql
CREATE TABLE sales
USING DELTA
LOCATION '/mnt/data/sales';
```

```text
Data is stored at an external location.
Dropping the table removes table metadata but keeps the files.
```

This is a very common exam distinction.

## 17. SQL Table, View, and Schema Syntax

Create table as select:

```sql
CREATE TABLE sales_summary AS
SELECT * FROM sales;
```

Create schema and use schema:

```sql
CREATE SCHEMA sales;
USE sales;
```

Create view:

```sql
CREATE VIEW my_view AS
SELECT * FROM sales;
```

Create temporary views from a DataFrame:

```python
df.createOrReplaceTempView("my_view")  # session-scoped
df.createGlobalTempView("my_view")     # available across sessions in the same Spark app
```

Drop table:

```sql
DROP TABLE sales;
```

Drop behavior:

```text
managed table  -> data deleted
external table -> files kept
```

## 18. ALTER TABLE and Constraints

Rename a table:

```sql
ALTER TABLE old_name
RENAME TO new_name;
```

Add columns:

```sql
ALTER TABLE sales
ADD COLUMNS (
    region STRING
);
```

Change a column comment:

```sql
ALTER TABLE sales
ALTER COLUMN amount COMMENT 'sales amount';
```

Set table properties:

```sql
ALTER TABLE sales
SET TBLPROPERTIES (
    'quality' = 'gold'
);
```

Set a NOT NULL constraint:

```sql
ALTER TABLE sales
ALTER COLUMN amount SET NOT NULL;
```

Important restrictions:

```text
Cannot directly reorder columns in many Spark/Delta cases.
SET NOT NULL fails if existing rows contain NULL.
DROP COLUMN is supported only in newer Delta versions; historically it was restricted.
```

## 19. Unity Catalog

Unity Catalog hierarchy:

```text
Metastore
  Catalog
    Schema
      Table/View
```

Three-level namespace:

```sql
CREATE TABLE main.sales.orders (...);
```

Meaning:

```text
main   -> catalog
sales  -> schema
orders -> table
```

Common governance commands to recognize:

```sql
USE CATALOG main;
USE SCHEMA sales;
GRANT SELECT ON TABLE sales TO `user@example.com`;
REVOKE SELECT ON TABLE sales FROM `user@example.com`;
```

## 23. High-Probability Trick Topics

These are disproportionately testable:

| Topic | Exam trap |
| --- | --- |
| `to_date` vs `date_format` | date type vs string type |
| `explode` vs `explode_outer` | null behavior |
| `union` vs `unionByName` | schema order vs column names |
| `rank` vs `dense_rank` | skipped numbers vs no gaps |
| `regexp_replace` vs `translate` | regex pattern vs character replacement |
| `collect_list` vs `collect_set` | duplicates kept vs removed |
| `coalesce` | first non-null value |
| `cast` | type conversion |
| `isin` | SQL `IN` equivalent |
| `when` / `otherwise` | conditional column logic |
| `expr` / `selectExpr` | SQL expressions inside PySpark |
| `substring` | starts at 1 |
| `element_at` | 1-based indexing |
| `collect()` | brings all data to driver |
| `count()` | triggers full computation |
| broadcast join | small table sent to executors |
| lazy evaluation | transformations wait until actions |
| shuffle | expensive network/disk operation |
