# Databricks Data Engineer Associate: Python, SQL, and Spark Cheat Sheet

Use this as a compact review map for the DE Associate exam. The exam likes syntax, return types, table behavior, and "what triggers execution?" questions.

## 1. Spark Session, Reading, and Writing

Create a Spark session:

```python
# Import the entry point used to create or access a Spark application.
from pyspark.sql import SparkSession

# Build or reuse a SparkSession named "MyApp".
spark = SparkSession.builder \
    .appName("MyApp") \
    .getOrCreate()
```

Read common file formats:

```python
# Read a CSV file and use the first row for column names.
df = spark.read.csv("/path/file.csv", header=True, inferSchema=True)

# Read semi-structured JSON data.
df = spark.read.json("/path/file.json")

# Read columnar Parquet data.
df = spark.read.parquet("/path/file.parquet")

# Read a Delta table from a storage path.
df = spark.read.format("delta").load("/path")
```

Write common outputs:

```python
# Overwrite existing Parquet output at the path.
df.write.mode("overwrite").parquet("/output")

# Save the DataFrame in Delta format.
df.write.format("delta").save("/output")

# Save as a managed table unless an external location is specified.
df.write.saveAsTable("sales")  # managed table by default
```

## 2. DataFrame Basics

Core transformations:

```python
# Import column helpers used by DataFrame transformations.
from pyspark.sql.functions import col, lit

# Select only the listed columns.
df.select("name", "age")

# Filter rows with a DataFrame column expression.
df.filter(df.age > 18)

# Filter rows with the col() helper, which is preferred for complex logic.
df.filter(col("age") > 18)

# Filter rows with a SQL-style condition string.
df.where("age > 18")

# Add or replace a column using an expression.
df.withColumn("age_plus_1", col("age") + 1)

# Add a constant literal value as a new column.
df.withColumn("country", lit("Canada"))

# Rename a column without changing its values.
df.withColumnRenamed("old", "new")

# Remove a column from the DataFrame.
df.drop("column_name")

# Return unique values for one selected column.
df.select("country").distinct()

# Remove duplicate rows across all columns.
df.dropDuplicates()

# Remove duplicates based only on the email column.
df.dropDuplicates(["email"])

# Sort rows by age in ascending order by default.
df.orderBy("age")

# Sort rows by age ascending explicitly.
df.orderBy(col("age").asc())

# Sort rows by age descending.
df.orderBy(col("age").desc())
```

Classic valid vs invalid traps:

```python
# Use col() like a function to reference a Spark column.
col("age")                 # correct

# Bracket syntax is invalid for the col helper.
col["age"]                 # wrong

# Replace or create a column with withColumn().
df.withColumn("age", lit(5))  # correct

# DataFrame columns cannot be assigned with Python attribute syntax.
df.age = 5                    # wrong

# Build conditional column logic with when().
when(col("x") > 5, "high")    # correct

# Python if statements do not work on distributed Spark column expressions.
if col("x") > 5:              # wrong for column logic
    ...
```

Use bitwise operators for column conditions:

```python
# Use & for AND between Spark column expressions.
(col("age") > 18) & (col("country") == "CA")  # AND

# Use | for OR between Spark column expressions.
(col("age") > 18) | (col("vip") == True)      # OR

# Use ~ to negate a Spark boolean column expression.
~col("deleted")                               # NOT
```

Do not use Python `and` / `or` with Spark column expressions.

## 4. Type Conversion and Datetime

Cast columns:

```python
# Convert age to an integer column.
df.withColumn("age", col("age").cast("int"))

# Convert salary to a double column.
df.withColumn("salary", col("salary").cast("double"))

# Convert created_at to a timestamp column.
df.withColumn("created_ts", col("created_at").cast("timestamp"))
```

Parse and format dates/timestamps:

```python
# Parse a string column into a date using the given pattern.
df.withColumn("date", to_date(col("date_str"), "yyyy-MM-dd"))

# Parse a string column into a timestamp using the given pattern.
df.withColumn("ts", to_timestamp(col("ts_str"), "yyyy-MM-dd HH:mm:ss"))

# Format a timestamp as a string.
df.withColumn("formatted", date_format(col("ts"), "yyyy/MM/dd"))
```

Extract or calculate date parts:

```python
# Extract the year from a timestamp.
year(col("ts"))

# Extract the month from a timestamp.
month(col("ts"))

# Extract the day of month from a timestamp.
dayofmonth(col("ts"))

# Extract the hour from a timestamp.
hour(col("ts"))

# Return the current timestamp.
current_timestamp()

# Return the current date.
current_date()

# Calculate the number of days between end and start.
datediff(col("end"), col("start"))  # returns days

# Add seven days to a date.
date_add(col("date"), 7)

# Subtract seven days from a date.
date_sub(col("date"), 7)
```

Datetime format tokens:

```text
yyyy -> year
MM   -> month
dd   -> day
HH   -> hour, 24-hour clock
mm   -> minute
ss   -> second
```

Return-type trap:

```text
to_date()      -> date
to_timestamp() -> timestamp
date_format()  -> string
```

## 5. Null Handling and Conditional Logic

Check, fill, drop, and replace nulls:

```python
# Check whether name is null.
col("name").isNull()

# Check whether name is not null.
col("name").isNotNull()

# Fill all compatible null values with 0.
df.na.fill(0)

# Fill nulls in age with 0.
df.na.fill({"age": 0})

# Drop rows that contain null values.
df.na.drop()

# Return nickname when present, otherwise name.
coalesce(col("nickname"), col("name"))  # first non-null value

# Replace UNKNOWN values with null.
df.na.replace("UNKNOWN", None)

# fillna() is the DataFrame alias for na.fill().
df.fillna(0)                            # alias for na.fill()

# Replace M values with Male at the DataFrame level.
df.replace("M", "Male")                 # DataFrame-level replacement
```

Use `when` / `otherwise` for conditional columns:

```python
# Create a PASS/FAIL status column based on score.
df.withColumn(
    "status",
    when(col("score") >= 60, "PASS").otherwise("FAIL")
)

# Chain when() clauses to create grade buckets.
df.withColumn(
    "grade",
    when(col("score") >= 90, "A")
    .when(col("score") >= 80, "B")
    .otherwise("C")
)
```

## 6. String Functions

Common string transforms:

```python
# Convert name to uppercase.
upper(col("name"))

# Convert name to lowercase.
lower(col("name"))

# Remove spaces from the start and end of name.
trim(col("name"))

# Join first and last directly together.
concat(col("first"), col("last"))

# Join year and month with a dash separator.
concat_ws("-", col("year"), col("month"))

# Remove dashes from phone values using regex replacement.
regexp_replace(col("phone"), "-", "")

# Split email into an array around @.
split(col("email"), "@")          # returns array

# Check whether email contains the given text.
col("email").contains("@gmail.com")

# Extract the first three characters; Spark substring starts at 1.
substring(col("id"), 1, 3)        # starts at 1, not 0
```

More string functions that can appear:

```python
# Replace each matching character independently.
translate(col("phone"), "-", "")          # character-by-character replacement

# Extract the first regex capture group after @.
regexp_extract(col("email"), r"@(.*)", 1) # extract regex group

# Capitalize each word in the name.
initcap(col("name"))                      # john smith -> John Smith

# Return the number of characters.
length(col("name"))

# Return the position of @ in the string.
instr(col("email"), "@")

# Left-pad id with zeros until it has length 5.
lpad(col("id"), 5, "0")                   # 12 -> 00012

# Right-pad id with zeros until it has length 5.
rpad(col("id"), 5, "0")

# Return text before the first @.
substring_index(col("email"), "@", 1)     # text before @

# Check whether email begins with admin.
col("email").startswith("admin")

# Check whether email ends with .com.
col("email").endswith(".com")

# Match email using a SQL LIKE pattern.
col("email").like("%gmail%")

# Check whether country is one of the listed values.
col("country").isin("CA", "US", "UK")     # SQL IN equivalent

# Check whether age is within the inclusive range.
col("age").between(18, 65)
```

Important comparison:

```text
translate()      -> character substitution
regexp_replace() -> regex pattern replacement
```

## 10. Joins and Broadcast

Common joins:

```python
# Join DataFrames where id matches, keeping only matched rows.
df1.join(df2, on="id", how="inner")

# Join DataFrames where id matches, keeping all left-side rows.
df1.join(df2, on="id", how="left")
```

Broadcast join:

```python
# Import broadcast to mark a small DataFrame for broadcast joins.
from pyspark.sql.functions import broadcast

# Broadcast df2 so it is sent to executors for the join.
df1.join(broadcast(df2), "id")
```

Exam model:

```text
Broadcast join -> small table sent to executors
Benefit        -> avoids shuffle on the large table
Risk           -> broadcasting a large table can cause memory issues
```

## 11. Window Functions

Define a window:

```python
# Import Window to define partition and ordering rules.
from pyspark.sql.window import Window

# Import row_number for ranking rows inside a window.
from pyspark.sql.functions import row_number

# Partition by department and order each department by descending salary.
windowSpec = Window.partitionBy("dept").orderBy(col("salary").desc())

# Add a rank column using row_number over the window.
df.withColumn("rank", row_number().over(windowSpec))
```

Common window functions:

```python
# Assign consecutive numbers within each window partition.
row_number().over(windowSpec)

# Rank rows, leaving gaps after ties.
rank().over(windowSpec)        # can skip numbers

# Rank rows without gaps after ties.
dense_rank().over(windowSpec)  # no gaps

# Return the previous row's salary within the window.
lag("salary", 1)               # previous row

# Return the next row's salary within the window.
lead("salary", 1)              # next row
```

## 12. Transformations, Actions, Shuffle, and Caching

Transformations are lazy:

```python
# Choose columns without triggering execution by itself.
select()

# Keep rows that match a condition without triggering execution by itself.
filter()

# Join DataFrames without triggering execution by itself.
join()

# Prepare grouped data for aggregation without triggering execution by itself.
groupBy()

# Add or replace a column without triggering execution by itself.
withColumn()
```

Actions trigger execution:

```python
# Print rows to the notebook or console and trigger execution.
show()

# Bring all rows back to the driver and trigger execution.
collect()

# Count rows and trigger execution.
count()

# Return the first n rows and trigger execution.
take()

# Return the first row and trigger execution.
first()

# Write output data and trigger execution.
write()
```

Narrow transformations do not require a shuffle:

```python
# Select columns without moving data between partitions.
select()

# Filter rows without moving data between partitions.
filter()

# Add or replace a column without moving data between partitions.
withColumn()
```

Wide transformations usually require a shuffle:

```python
# Aggregate by key and usually shuffle data across partitions.
groupBy()

# Join data and often shuffle unless optimized, such as by broadcast.
join()

# Remove duplicates and shuffle to compare values across partitions.
distinct()

# Globally sort data and usually shuffle.
orderBy()
```

Shuffle is expensive because it involves network I/O, possible disk spill, and serialization.

Cache and persist:

```python
# Mark this DataFrame to be cached after an action materializes it.
df.cache()

# Import storage levels for explicit persistence choices.
from pyspark import StorageLevel

# Persist the DataFrame in memory and spill to disk if needed.
df.persist(StorageLevel.MEMORY_AND_DISK)
```

Exam model:

```text
cache()   -> persist using the default storage level
persist() -> lets you choose the storage level
```

## 13. Driver, Executors, and OOM Patterns

Driver responsibilities:

```text
query planning
DAG generation
task scheduling
collecting results for driver-side actions
```

Executor responsibilities:

```text
actual computation
storing partitions
executing tasks
```

Driver OOM is commonly caused by:

```python
# Pulling all rows to the driver can exhaust driver memory.
collect()

# Converting to pandas pulls data to the driver and can exhaust memory.
toPandas()

# Broadcasting a table that is too large can exhaust driver or executor memory.
broadcast(large_df)
```

Executor OOM is commonly caused by:

```text
huge partitions
skewed joins
insufficient executor memory
```

## 14. Partitioning and Performance

Partition data when large datasets are frequently filtered by the partition column:

```python
# Write output partitioned into country folders.
df.write.partitionBy("country")
```

Change DataFrame partitions:

```python
# Increase or decrease to exactly 10 partitions with a shuffle.
df.repartition(10)  # causes shuffle
```

```text
repartition() -> can increase or decrease partitions, full shuffle
coalesce()    -> usually reduces partitions, less shuffle
```

## 15. Delta Lake Essentials

Delta Lake is a storage layer that provides ACID transactions, schema enforcement, and time travel.

Create a Delta table:

```sql
-- Create a managed Delta table named sales.
CREATE TABLE sales (
    id INT,
    amount DOUBLE
)
USING DELTA;
```

Convert Parquet to Delta:

```python
# Import DeltaTable utilities for Delta Lake operations.
from delta.tables import DeltaTable

# Convert an existing Parquet dataset at /path into Delta format.
DeltaTable.convertToDelta(
    spark,
    "parquet.`/path`"
)
```

Time travel:

```sql
-- Query the sales table as it existed at Delta version 3.
SELECT * FROM sales VERSION AS OF 3;

-- Query the sales table as it existed at a specific timestamp.
SELECT * FROM sales TIMESTAMP AS OF '2025-01-01';
```

Modify Delta tables:

```sql
-- Delete invalid negative sales rows.
DELETE FROM sales WHERE amount < 0;

-- Increase Canadian sales amounts by 10 percent.
UPDATE sales
SET amount = amount * 1.1
WHERE country = 'CA';
```

Merge / upsert:

```sql
-- Upsert source rows into target by matching on id.
MERGE INTO target t
USING source s
ON t.id = s.id

-- Update all target columns when a matching id exists.
WHEN MATCHED THEN
  UPDATE SET *

-- Insert all source columns when no matching id exists.
WHEN NOT MATCHED THEN
  INSERT *;
```

Clean and optimize:

```sql
-- Remove old unreferenced files older than 168 hours.
VACUUM sales RETAIN 168 HOURS;

-- Compact small files in the sales table.
OPTIMIZE sales;

-- Cluster related customer_id values to improve data skipping.
OPTIMIZE sales ZORDER BY (customer_id);
```

Exam model:

```text
VACUUM  -> removes old unused data files
OPTIMIZE -> compacts small files
ZORDER  -> improves data skipping for selected columns
```

## 16. Managed vs External Tables

Managed table:

```sql
-- Create a managed table; Databricks controls the table files.
CREATE TABLE sales (...);
```

```text
Databricks manages metadata and underlying files.
Dropping the table deletes the data.
```

External table:

```sql
-- Create an external Delta table over files at the given location.
CREATE TABLE sales
USING DELTA
LOCATION '/mnt/data/sales';
```

```text
Data is stored at an external location.
Dropping the table removes table metadata but keeps the files.
```

This is a very common exam distinction.
