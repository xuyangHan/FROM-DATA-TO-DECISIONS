# 3.4 Deduplication and aggregates

## Exam objective

> Perform data deduplication operations and aggregate operations on DataFrames, such as count, approximate count distinct, and mean, summary.

## Must know

- **`dropDuplicates()`** without arguments removes only exact duplicate rows across all columns.
- **`dropDuplicates(["key"])`** keeps one row per key subset. The kept row is not guaranteed to be the newest or best unless you define an ordering separately.
- **Deduplication key** should match the business grain: `event_id`, `order_id`, or `customer_id + effective_date`.
- **`groupBy(...).agg(...)`** is the main pattern for DataFrame aggregations.
- **`count("*")`** counts rows. SQL `COUNT(col)` ignores nulls in that column.
- **`avg` / `mean`** ignore null values.
- **`countDistinct`** is exact but expensive on high-cardinality data.
- **`approx_count_distinct`** is faster and approximate, useful for large cardinality estimates.
- **`summary()`** and **`describe()`** are quick profiling helpers, not usually production metric tables.

## Deduplication decision table

| Goal | Method | Watch out |
|------|--------|-----------|
| Remove exact repeated rows | `dropDuplicates()` | Does not remove same key with changed attributes |
| Keep one row per business key | `dropDuplicates(["id"])` | Kept row is arbitrary |
| Keep latest row per key | Window with `row_number()` | Requires reliable ordering column |
| Deduplicate streaming data | `dropDuplicates` with watermark when event time matters | Needs state and checkpoint |
| Find duplicate keys | `groupBy(key).count().filter("count > 1")` | Useful before joins |

## PySpark / API essentials

```python
from pyspark.sql.functions import approx_count_distinct, avg, col, count, countDistinct, max as max_, sum as sum_

events = spark.table("main.bronze.clickstream_raw")

# Exact full-row duplicates only.
unique_rows = events.dropDuplicates()

# Business-key deduplication.
deduped = events.dropDuplicates(["event_id"])

# Find duplicate keys for investigation.
duplicate_event_ids = (
    events
    .groupBy("event_id")
    .agg(count("*").alias("row_count"))
    .filter(col("row_count") > 1)
)
```

## Keep latest row per key

Use a window when the question implies "latest", "most recent", or "highest version".

```python
from pyspark.sql.window import Window
from pyspark.sql.functions import row_number

w = Window.partitionBy("customer_id").orderBy(col("updated_at").desc())

latest_customers = (
    spark.table("main.bronze.customers_raw")
    .withColumn("rn", row_number().over(w))
    .filter(col("rn") == 1)
    .drop("rn")
)
```

## Aggregation examples

```python
orders = spark.table("main.silver.orders")

daily_metrics = (
    orders
    .groupBy("order_date")
    .agg(
        count("*").alias("order_count"),
        countDistinct("customer_id").alias("exact_customers"),
        approx_count_distinct("customer_id").alias("approx_customers"),
        sum_("amount").alias("revenue"),
        avg("amount").alias("avg_order_amount"),
        max_("amount").alias("max_order_amount")
    )
)

daily_metrics.write.format("delta").mode("overwrite").saveAsTable("main.gold.daily_order_metrics")
```

```python
# Quick profiling, useful in notebooks.
orders.select("amount", "discount").summary().show()
orders.describe("amount").show()
```

## SQL (exam)

```sql
-- Find duplicate business keys.
SELECT order_id, COUNT(*) AS row_count
FROM main.bronze.orders_raw
GROUP BY order_id
HAVING COUNT(*) > 1;
```

```sql
-- Aggregate silver data for gold.
CREATE OR REPLACE TABLE main.gold.daily_order_metrics AS
SELECT
  order_date,
  COUNT(*) AS order_count,
  COUNT(DISTINCT customer_id) AS exact_customers,
  APPROX_COUNT_DISTINCT(customer_id) AS approx_customers,
  SUM(amount) AS revenue,
  AVG(amount) AS avg_order_amount
FROM main.silver.orders
GROUP BY order_date;
```

```sql
-- Keep latest row per customer.
CREATE OR REPLACE TABLE main.silver.customers AS
SELECT * EXCEPT (rn)
FROM (
  SELECT
    *,
    ROW_NUMBER() OVER (PARTITION BY customer_id ORDER BY updated_at DESC) AS rn
  FROM main.bronze.customers_raw
)
WHERE rn = 1;
```

## Count behavior

| Expression | Meaning |
|------------|---------|
| DataFrame `df.count()` | Count all rows; triggers a Spark job |
| PySpark `count("*")` | Count all rows in each group |
| SQL `COUNT(*)` | Count all rows |
| SQL `COUNT(column)` | Count non-null values in that column |
| `countDistinct(column)` | Exact number of unique non-null values |
| `approx_count_distinct(column)` | Approximate unique count, faster at scale |

## UI / product surfaces

- **Notebook display / data profile**: quick column stats, null counts, and value distributions.
- **Spark UI / Query Profile**: aggregation usually causes shuffle when grouped by key.
- **Catalog Explorer**: inspect row counts, table history, lineage, and comments for metric tables.

## Exam traps

- Using `dropDuplicates(["id"])` when the requirement says keep the latest record.
- Using exact `countDistinct` when the question asks for a scalable approximate estimate.
- Forgetting that `avg` ignores nulls, so missing values do not behave like zero.
- Confusing row count after `explode` with count of original entities.
- Performing a groupBy aggregation before cleaning invalid keys or bad amounts.
- Calling `df.count()` repeatedly in production code just for logging; it scans data each time.
